# 웹 UI 테스트 가이드 (SOP)

웹 페이지의 디자인, 반응성 및 Supabase 연동 기능을 검증하기 위한 절차입니다.

## 1. 목적
- 로컬 환경에서 `public/index.html`이 올바르게 렌더링되는지 확인합니다.
- 이메일 구독 폼의 유효성 검사 및 Supabase 전송 로직이 정상 동작하는지 테스트합니다.

## 2. 테스트 환경 준비
로컬 개발 서버를 실행하여 브라우저에서 페이지를 확인할 수 있도록 합니다.

```bash
# 로컬 개발 서버 시작 (Python 내장 서버 이용)
python execution/dev_server.py
```
- 서버가 실행되면 브라우저에서 `http://localhost:8000`으로 접속합니다.

## 3. 수동 테스트 케이스
다음 시나리오를 수행하여 UI/UX 및 기능을 검증합니다.

| 테스트 단계 | 입력값 | 예상 결과 |
| :--- | :--- | :--- |
| **1. 초기 렌더링** | N/A | 프리미엄 테마(다크 모드, 그라데이션)가 정상적으로 표시되어야 함 |
| **2. 유효하지 않은 이메일** | `test` (잘못된 형식) | "유효한 이메일 주소를 입력해 주세요." 에러 메시지 표시 |
| **3. 빈 이메일 입력** | (빈칸) | 에러 메시지 표시 |
| **4. 유효한 이메일 구독** | `your-email@example.com` | "구독 확인" 알림창 표시 및 "구독 완료!" 메시지 표시 |
| **5. Supabase 연동 확인** | (구독 후) | Supabase 대시보드 또는 로그를 통해 `subscribers` 테이블에 이메일이 추가되었는지 확인 |

## 4. 자동화 테스트 (선택 사항)
브라우저 자동화 툴(예: Playwright)을 사용하여 UI 회귀 테스트를 수행할 수 있습니다. (현재는 수동 테스트를 권장합니다.)

## 5. 피드백 및 수정
- 디자인에 미비한 점이 발견되면 `public/index.html`의 CSS 섹션을 수정합니다.
- 로직 오류가 발견되면 `<script>` 블록을 수정합니다.
