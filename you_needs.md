# 프로젝트 기술 검토 및 개선 제안 (Tester's Perspective)

현재 프로젝트(StockTradingTop10)를 테스터 및 기술 검토자의 시각으로 분석한 결과, 시스템의 안정성, 확장성, 그리고 사용자 경험(UX) 측면에서 개선이 필요한 사항들을 정리하였습니다.

---

## 1. 백엔드 및 자동화 배치 (Background Automation)

### 🔴 [중요] 중복 로직의 모듈화 (DRY 원칙)
- **현황**: `get_stock_data.py`, `get_news.py`, `save_to_db.py` 등 거의 모든 실행 스크립트에서 `check_lock`, `remove_lock`, `get_supabase_client` 코드가 중복되어 있습니다.
- **제안**: `execution/utils.py` 또는 `execution/core.py`를 하나 만들어 공통 기능을 분리하세요. 코드가 간결해지고 유지보수가 수월해집니다.

### 🟠 [신뢰성] 뉴스 스크레이핑의 불안정성
- **현황**: 네이버 뉴스를 고정된 CSS 셀렉터로 가져오고 있습니다. 포털 사이트의 UI가 변경되면 배치가 즉시 중단될 수 있습니다.
- **제안**:
  - 셀렉터가 실패할 경우를 대비한 **Back-up 셀렉터** 또는 **RSS 피드** 방식을 병행하세요.
  - 뉴스 본문 추출(`fetch_article_content`) 시 타임아웃 처리가 되어있으나, 특정 사이트에서 무한 대기가 발생할 수 있으므로 더 엄격한 관리가 필요합니다.

### 🟡 [모니터링] 상세 로그 데이터베이스 기록 활성화
- **현황**: `main.py`에서 `log_content` 필드가 DB 컬럼 누락으로 인해 주석 처리되어 있습니다.
- **제안**: Supabase 테이블(`execution_logs`, `batch_summary`)에 `log_content` (TEXT 타입) 컬럼을 추가하고 주석을 해제하세요. 관리 페이지에서 발생한 에러를 즉시 확인할 수 있게 됩니다.

---

## 2. 데이터베이스 및 보안 (DB & Security)

### 🟢 [보안] RLS(Row Level Security) 정책 재검토
- **현황**: 대부분의 정책이 `service_role`에게만 허용되어 있거나 `public`에 대해 `INSERT`만 허용되어 있습니다.
- **제안**: 사용자 페이지에서 조회하는 `companies`, `news_articles`, `stock_analysis` 테이블에 대해 `public` 읽기 전용 정책(`SELECT`)이 명확히 설정되어 있는지 다시 확인하세요.

### 🟡 [최적화] 인덱스(Index) 추가
- **현황**: 데이터가 쌓일수록 `symbol`이나 `company_id` 기반의 조회가 느려질 수 있습니다.
- **제안**: `symbol`, `company_id`, `published_at` 컬럼에 인덱스를 생성하여 렌더링 속도를 개선하세요.

---

## 3. 프론트엔드 및 사용자 경험 (Frontend & UX)

### 🟠 [성능] 데이터 호출 효율화
- **현황**: `index.js`의 `loadNews` 함수에서 회사별 AI 분석 정보를 가져오기 위해 루프 안에서 개별적으로 `sbGet`을 호출하고 있습니다. (N+1 문제)
- **제안**: `stock_analysis` 데이터를 회사 ID 리스트(`in` 쿼리)를 사용하여 한 번에 배치(Batch)로 가져오도록 수정하세요. 페이지 로딩 속도가 획기적으로 빨라집니다.

### 🟢 [UX] 로딩 및 에러 상태의 시각화
- **현황**: 데이터를 불러오는 동안 "로딩 중" 텍스트만 표시되거나, 실패 시 단순한 얼럿창만 뜹니다.
- **제안**:
  - 스켈레톤 UI(Skeleton UI)를 도입하여 로딩 중에도 레이아웃이 깨지지 않게 하세요.
  - 네트워크 오류 발생 시 "다시 시도" 버튼을 제공하여 사용자 이탈을 방지하세요.

### 🔵 [기능 확장] 뉴스 검색 및 필터링
- **현황**: 현재는 고정된 리스트만 보여주고 있습니다.
- **제안**: 사용자가 관심 있는 종목을 검색하거나, '상승 우세' 종목만 모아보는 필터링 기능을 추가하면 서비스의 활용도가 높아집니다.

---

## 4. 시큐어 코딩 및 관리 (Secure Coding)

### 🔴 [보안] API 키 노출 관리
- **현황**: `JS` 파일에 Supabase URL과 Key가 하드코딩되어 있습니다. (Public Key이므로 치명적이진 않지만 관리가 필요함)
- **제안**: 환경 변수를 통해 주입하거나, 민감한 로직은 Supabase Edge Functions를 활용하여 백엔드에서 처리하도록 전환을 고려하세요.

---

**결론**: 전체적인 서비스의 흐름은 매우 견고하지만, **코드의 재사용성(모듈화)**과 **프론트엔드 데이터 호출 최적화**가 현재 시점에서 가장 시급한 과제입니다.
